**Developer Productivity Intelligence**

DevPulse is a full-stack MVP that helps individual software developers move beyond raw productivity metrics toward genuine understanding and action. Instead of showing numbers in isolation, DevPulse interprets what those numbers mean in context and suggests practical next steps вЂ” powered by Groq's Llama 3.3 model.

Built as part of an intern assignment focused on product thinking, full-stack engineering, data handling, and responsible AI use.

---

## The Problem It Solves

Developers and managers can already see metrics like lead time, cycle time, and bug rate in various tools. The real problem is that **metrics alone don't explain what is happening or what to do next**. DevPulse bridges that gap by combining metric display with AI-generated interpretation and actionable coaching.

---

## Features

### IC View (Individual Contributor)
- Select any developer and month from the available data
- View all 5 productivity metrics in colour-coded cards
- See a pattern badge summarising the developer's overall health signal
- Click **Get AI Insight** to receive a plain-English interpretation and 2 specific next steps generated by Llama 3.3 via Groq

### Manager View
- Lightweight summary table showing team-level signals across all managers and months
- At-a-glance health indicators for each team

## The 5 Metrics

| Metric | Definition |
|---|---|
| **Lead Time for Changes** | Average days from PR opened to successful production deployment |
| **Cycle Time** | Average days from issue moved to In Progress to marked Done |
| **Bug Rate** | Escaped production bugs Г· issues completed in the month |
| **Deployment Frequency** | Count of successful production deployments in the month |
| **PR Throughput** | Count of merged pull requests in the month |

## Data Model

The Excel workbook acts as a multi-table data source, similar to how separate systems like Jira, GitHub, and a CI/CD pipeline would feed into an analytics layer.

| Sheet | What It Represents |
|---|---|
| `Dim_Developers` | Developer profiles вЂ” name, team, level, manager |
| `Fact_Jira_Issues` | Issue-level data with cycle time |
| `Fact_Pull_Requests` | PR-level data with review wait and merge time |
| `Fact_CI_Deployments` | Deployment records with lead time |
| `Fact_Bug_Reports` | Post-release bugs with escaped-to-prod flag |
| `Metric_Examples` | Pre-aggregated metrics per developer per month вЂ” primary data source for the app |
| `Manager_Sample_View` | Pre-aggregated team-level signals per manager per month |

The app reads the `Metric_Examples` sheet as its primary metric source rather than recalculating from raw tables, keeping the logic simple and interview-explainable.

## Getting Started

### Prerequisites
- Node.js v18 or above
- A free Groq API key вЂ” get one at [console.groq.com](https://console.groq.com)

### Installation

1. Clone the repository
```bash
git clone https://github.com/your-username/devpulse.git
cd devpulse
```

2. Install root dependencies
```bash
npm install
```

3. Install frontend dependencies
```bash
cd frontend && npm install && cd ..
```

4. Set up your environment variables
```bash
cp .env.example .env
```
Open `.env` and add your Groq API key:
```
GROQ_API_KEY=your_groq_api_key_here
```

5. Place the data file
Make sure the Excel workbook is placed at:
```
data/intern_assignment_support_pack_dev_only_v3.xlsx
```

### Running the App

Run both frontend and backend together:
```bash
npm run dev
```

Or run them separately:
```bash
npm run dev:backend    # Express server on port 3001
npm run dev:frontend   # Vite dev server on port 5173
```

Then open [http://localhost:5173](http://localhost:5173) in your browser.

## How the AI Layer Works

When a user clicks **Get AI Insight**, the backend:

1. Fetches the developer's metrics for the selected month
2. Builds a prompt containing the developer's name, team, pattern hint, and all 5 metric values
3. Sends that prompt to Groq's `llama-3.3-70b-versatile` model with a system prompt that instructs it to act as a developer productivity coach
4. Parses the structured JSON response
5. Returns an `interpretation` (under 60 words) and `nextSteps` (exactly 2 actionable sentences) to the frontend

Groq was chosen over other providers because it offers a generous free tier and extremely fast inference, which makes the insight response feel near-instant during the demo.
